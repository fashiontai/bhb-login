# BHB 登录前端 PRD

## 1. 文档信息

- 文档版本: v1.0
- 编写日期: 2026-07-05
- 适用项目: `bhb-login`
- 适用范围: `apps/web` 前端应用与 `packages/ui` 公共 UI 组件
- 需求来源: 当前前端实现、用户提出的组件边界与中英文切换要求

## 2. 背景

`bhb-login` 是一个基于 Better Auth 的登录系统前端。当前仓库已从测试用例和脚手架页面转向真实登录业务，需要沉淀稳定的登录、注册、会话守卫、退出登录和中英文切换能力。

前端采用 monorepo 结构，业务页面位于 `apps/web`，可复用 UI 组件位于 `packages/ui`。本 PRD 约束前端需求边界，避免后续把业务逻辑放入公共组件包，或把通用 UI 重复实现到业务应用内。

## 3. 产品目标

1. 为用户提供清晰、可用、响应式的登录和注册体验。
2. 支持中文和英文界面切换，默认中文，并记住用户选择。
3. 保护需要登录访问的页面，未登录用户自动回到登录页。
4. 将业务组件和公共组件分层，保证后续扩展登录页、鉴权页和其他系统入口时可复用。
5. 保持前端实现可维护、可验证，并符合 Ultracite 代码规范。

## 4. 用户与使用场景

| 用户类型 | 场景 | 目标 |
| --- | --- | --- |
| 未登录用户 | 访问系统入口 | 可以登录已有账户，或切换到注册表单创建账户 |
| 新用户 | 没有账户 | 可以通过邮箱、密码、姓名注册账户 |
| 已登录用户 | 访问受保护页面 | 可以进入控制台并查看欢迎信息 |
| 双语用户 | 使用不同语言环境 | 可以在中文和英文之间切换，刷新后保留选择 |

## 5. 范围

### 5.1 本期范围

- 登录页壳层、品牌、页头、页脚。
- 邮箱密码登录表单。
- 姓名、邮箱、密码注册表单。
- 登录和注册之间的表单切换。
- 登录成功、注册成功后的控制台跳转。
- 受保护路由会话检查。
- 控制台欢迎信息和退出登录。
- 中文和英文切换。
- 中英文页面标题、描述、按钮、表单、校验错误、提示信息。
- 公共认证 UI 组件封装到 `packages/ui`。
- 登录/注册业务组件保留在 `apps/web`。

### 5.2 本期不包含

- Google OAuth 实际接入。
- 找回密码实际接入。
- 用户中心、权限管理、个人资料编辑等完整控制台能力。
- 多语言后台管理、远程文案配置或第三方 i18n 平台。
- 埋点统计、风控、人机校验、短信或邮件验证码。

## 6. 功能需求

### F-001 登录页布局

登录页需要提供统一认证壳层，包括页头、品牌、导航动作区、主体表单区域和页脚。

验收标准:

- AC-001: 用户访问 `/login` 时，默认看到登录表单。
- AC-002: 页面顶部展示品牌名称和导航动作。
- AC-003: 页面底部展示品牌、隐私政策、服务条款、Cookie 政策和版权信息。
- AC-004: 页面在桌面和移动端均可正常展示；移动端隐藏辅助导航，只保留语言切换。
- AC-005: 登录页公共布局由 `packages/ui/src/components/auth.tsx` 提供，业务页面通过 props 传入文案和动作。

### F-002 邮箱密码登录

用户可以通过邮箱和密码登录系统。

验收标准:

- AC-001: 登录表单包含邮箱和密码输入项。
- AC-002: 邮箱必须符合邮箱格式。
- AC-003: 密码长度至少 8 个字符。
- AC-004: 表单提交时调用 Better Auth 邮箱登录能力。
- AC-005: 登录成功后跳转到 `/dashboard`。
- AC-006: 登录失败时展示服务端返回的错误信息。
- AC-007: 提交中按钮展示当前语言对应的加载文案。

### F-003 注册账户

新用户可以通过姓名、邮箱和密码创建账户。

验收标准:

- AC-001: 用户点击登录页的创建账户入口后，切换到注册表单。
- AC-002: 注册表单包含姓名、邮箱、密码输入项。
- AC-003: 姓名至少 2 个字符。
- AC-004: 邮箱必须符合邮箱格式。
- AC-005: 密码长度至少 8 个字符。
- AC-006: 表单提交时调用 Better Auth 邮箱注册能力。
- AC-007: 注册成功后跳转到 `/dashboard`。
- AC-008: 注册失败时展示服务端返回的错误信息。
- AC-009: 用户可以从注册表单切回登录表单。

### F-004 中英文切换

用户可以在中文和英文之间切换界面语言。

验收标准:

- AC-001: 系统默认语言为中文 `zh-CN`。
- AC-002: 用户点击语言切换按钮后，界面切换到另一种语言。
- AC-003: 切换语言后，登录、注册、控制台、校验错误、Toast、页头、页脚文案同步变化。
- AC-004: 语言选择保存到 `localStorage`，刷新页面后保持用户选择。
- AC-005: 切换语言时同步更新 `document.documentElement.lang`。
- AC-006: 切换语言时同步更新页面标题和 description。
- AC-007: tRPC 查询错误 Toast 的重试按钮使用当前语言。

### F-005 受保护路由与控制台

系统需要保护登录后页面，未登录用户不能直接访问控制台。

验收标准:

- AC-001: 用户访问 `/dashboard` 前先通过 Better Auth 获取 session。
- AC-002: 没有 session 时自动重定向到 `/login`。
- AC-003: 有 session 时展示控制台页面。
- AC-004: 控制台展示当前用户姓名；没有姓名时展示当前语言对应的默认用户称呼。
- AC-005: 用户点击退出登录后调用 Better Auth 退出接口。
- AC-006: 退出成功后跳转到 `/login`。

### F-006 未接入能力提示

Google 登录和找回密码当前未完成后端/OAuth 配置，前端只提供明确提示，避免用户误以为功能已接入。

验收标准:

- AC-001: 点击 Google 登录按钮时展示“暂未配置 Google 登录”或英文对应提示。
- AC-002: 点击忘记密码时展示“暂未配置密码重置”或英文对应提示。
- AC-003: 未接入功能不发起无效网络请求。

### F-007 组件分层

前端组件必须按业务和通用 UI 分层。

验收标准:

- AC-001: 登录/注册表单、语言切换、业务路由留在 `apps/web`。
- AC-002: 通用认证壳层、认证卡片、字段、分割线、Google 按钮等 UI 组件放在 `packages/ui`。
- AC-003: `packages/ui` 不依赖 Better Auth、路由、业务文案状态或业务 API。
- AC-004: `apps/web` 通过 props 和组合方式使用 `packages/ui`。

## 7. 非功能需求

### 7.1 可访问性

- 表单字段必须有可关联的 label。
- 字段错误需要通过 `aria-invalid` 和 `aria-describedby` 暴露给辅助技术。
- 导航区域需要有当前语言的 `aria-label`。
- 语言切换按钮需要有可读的 `aria-label` 和 `title`。
- 图标仅装饰时需要设置 `aria-hidden`。

### 7.2 响应式

- 登录卡片最大宽度在桌面端保持可读，在移动端适配窄屏。
- 页头导航在移动端收敛，避免按钮挤压。
- 表单按钮、输入框高度稳定，避免状态变化造成布局跳动。

### 7.3 可维护性

- 所有前端可见文案集中在 `apps/web/src/i18n.tsx`。
- 新增文案必须同时补充中文和英文。
- 公共 UI 组件只提供默认文案和可传入 props，不承载业务状态。
- 代码需要通过 `pnpm run check`、`pnpm run check-types`、`pnpm run build`。

### 7.4 性能

- 语言切换使用本地状态和 `localStorage`，不依赖服务端请求。
- i18n 文案表保持轻量，不引入额外运行时国际化依赖。
- 当前阶段不要求按语言分包。

## 11. 关键交互流程

### 11.1 登录流程

1. 用户访问 `/login`。
2. 输入邮箱和密码。
3. 前端执行邮箱和密码校验。
4. 校验通过后调用 Better Auth 登录。
5. 成功后跳转 `/dashboard`。
6. 失败后展示错误 Toast。

### 11.2 注册流程

1. 用户在登录页点击创建账户。
2. 页面切换到注册表单。
3. 用户输入姓名、邮箱和密码。
4. 前端执行字段校验。
5. 校验通过后调用 Better Auth 注册。
6. 成功后跳转 `/dashboard`。
7. 失败后展示错误 Toast。

### 11.3 语言切换流程

1. 用户点击页头语言切换按钮。
2. 前端切换 `zh-CN` 和 `en-US`。
3. 文案、HTML lang、页面标题和 description 更新。
4. 当前语言写入 `localStorage`。
5. 用户刷新页面后恢复上一次选择。

### 11.4 受保护路由流程

1. 用户访问 `/dashboard`。
2. 路由守卫调用 Better Auth 获取 session。
3. 无 session 时跳转 `/login`。
4. 有 session 时渲染控制台。
5. 用户点击退出登录后，退出成功并回到 `/login`。

## 14. 风险与待确认

1. Google OAuth 尚未配置，需要确认是否接入、接入哪个 OAuth provider，以及回调地址。
2. 找回密码尚未配置，需要确认邮箱服务、重置链接有效期和安全策略。
3. 控制台当前仅为基础页，需要确认后续真实业务入口。
4. 隐私政策、服务条款、Cookie 政策当前为占位链接，需要确认正式 URL。
5. 英文品牌是否固定为 `BHB Login` 需要业务确认。
