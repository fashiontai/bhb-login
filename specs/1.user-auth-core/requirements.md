# user-auth-core — 需求规格

## 概述

沉淀稳定的邮箱密码登录/注册闭环：注册即可直接使用（暂不做邮箱验证与邮箱绑定），会话守卫、退出登录可靠，错误提示国际化，并移除与邮件绑定相关的占位。

## 项目信息

- 项目名: bhb-login
- 架构类型: monorepo（pnpm workspaces + Turborepo；前端 Vite + React + TanStack Router，后端 Hono + tRPC，鉴权 Better Auth，DB Prisma + PostgreSQL）

## 用户故事

- 作为未登录用户，我想用邮箱和密码登录，以便进入受保护的控制台。
- 作为新用户，我想用姓名、邮箱、密码注册并立即使用，无需邮箱验证。
- 作为已登录用户，我想安全退出，以便结束当前会话。
- 作为双语用户，我想看到中英文一致、可读的错误提示。

## 功能需求

1. [F-001] 邮箱 + 密码登录，成功后进入 `/dashboard`。
2. [F-002] 姓名 + 邮箱 + 密码注册，注册成功后自动建立会话并进入 `/dashboard`。
3. [F-003] 暂不启用邮箱验证与邮箱绑定：`emailAndPassword` 开启，注册后无需验证即可登录使用。
4. [F-004] 会话守卫：未登录访问 `/_auth/*`（含 `/dashboard`）自动重定向到 `/login`。
5. [F-005] 退出登录：清除服务端会话并跳转 `/login`。
6. [F-006] 登录/注册错误国际化映射：邮箱已存在、凭据无效、密码过短等错误以中英文 toast 呈现。
7. [F-007] 移除登录页「忘记密码」邮件重置占位（与暂不考虑邮箱绑定一致）；保留 Google 登录为「暂未配置」占位。

## 非功能需求

- 性能: 登录/注册请求交互无明显阻塞，提交态禁用按钮防重复提交。
- 安全: 密码至少 8 位；Cookie `httpOnly + secure + sameSite=none`（沿用现状）；不在前端日志输出凭据。
- 兼容性: 中英文界面文案一致；桌面与移动端布局可用。

## 验收标准

- [ ] [AC-001] 新用户可注册成功并直接进入 dashboard，无需邮箱验证。
- [ ] [AC-002] 已注册用户可用正确邮箱密码登录；错误凭据显示可读错误提示。
- [ ] [AC-003] 重复邮箱注册时显示「邮箱已被使用」类国际化提示，不产生 500。
- [ ] [AC-004] 未登录直接访问 `/dashboard` 被重定向到 `/login`。
- [ ] [AC-005] 点击退出登录后会话失效，再访问 `/dashboard` 会被重定向到 `/login`。
- [ ] [AC-006] 登录页不再出现「忘记密码」入口；Google 按钮点击提示「暂未配置」。

## 依赖

- Better Auth（`better-auth`，emailAndPassword）
- Prisma + PostgreSQL（现有 User/Session/Account 模型）
- `@bhb-login/env`（BETTER_AUTH_SECRET、DATABASE_URL、CORS_ORIGIN 等）
