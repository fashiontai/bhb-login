## 关键决策（2026-07-05 与产品确认）

- **2-3 语言记录**：落库到 `User.language` 并在下次登录后自动恢复界面语言。
- **2-2 修改密码**：登录态改密（当前密码 + 新密码），使用 Better Auth `changePassword`，**不走邮件重置**。
- **改密入口**：放在现有 `/dashboard` 页内的「修改密码」卡片。
- **登录页占位**：移除「忘记密码」邮件重置占位；保留 Google 登录为「暂未配置」占位。

## ID 编号约定

- 功能需求 / 任务 / 验收标准 ID **在单个 feature 内编号**，跨 feature 用 `{序号}.` 前缀区分。
- 例：`1.T-001` = 序号 1 这个 feature 的 T-001；`3.F-005` = 序号 3 的 F-005。
- **跨 feature 依赖**写全限定 ID，如 `2.T-002 依赖 1.T-001`。
